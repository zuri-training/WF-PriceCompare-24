ERD Documentations

ERD stands for entity relationship diagram, which is a type of diagram that can help one create a clear visualization of one's database design. ERDsm also called ER models, are used to describe data and how pieces of data interact with one another. For this reason, ERDs are extremely important in database design and projects that require a clear structure of all data. By applying this standard, we ca easily understand the structure of a database or the information collected within a system.

Drawing entity relationship diagrams is time-consuming, but online diagramming software e.g gliffy, lucidchart etc. contains tools and templates to help diagramming easier.

The database store all authenticated users and distinguishes who uses the website, the time of usage, the reason for usage and other specific data. The ErD could be exported to MySQL server,sqlite,Oracle or any DBMS in JSON format and could be integrated into any database of any framework.
